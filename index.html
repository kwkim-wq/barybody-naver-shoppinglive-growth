요인별 영향력, 어떻게 측정하나

KPI는 인크리멘탈 매출(라이브 안 한 날 평균 대비 순증). 유입 지표(방문·동접·알림)와 전환 지표(전환율·매출)를 나눠 본다.

← 좌우로 스크롤

요인측정측정 방법
A-1 시간대간접그 자체로는 성과가 안 나옴 → 회차별 비교(오전 회차 평균 vs 야간 회차 평균)로 효과 파악
A-2 쇼호스트직접우리가 호스트를 직접 섭외·지정 가능 → 회차별로 호스트를 바꿔 전환율·체류·인크리멘탈 비교 + 정성 평가
A-3 상품 구성직접라이브에서 판매한 상품별 매출·전환율을 스마트스토어 상품 데이터로 직접 확인 (구성별 비교)
B-1 플친직접라이브 URL에 nt_keyword=kakao_friend_날짜 → 사용자정의채널 리포트에서 플친 유입·전환 분리
B-2 토스 이벤트직접토스 유입 링크에 식별 파라미터(nt_keyword=toss_날짜) → 단, 체리피커 비중 커서 전환율로 진성 판별
B-3 GFA 라이브 소재직접사용자정의채널의 nt_source=gfa·소재ID로 유입·전환 측정 + GFA 광고비로 ROAS

유입 vs 전환을 나눠서 본다

B그룹(플친·토스·GFA)은 "사람을 데려오는" 요인이라 유입은 쉽게 늘린다. 하지만 전환율이 같이 유지·상승해야 진짜 매출 기여다. 유입만 늘고 전환율이 떨어지면 허수(특히 토스 체리피커). A그룹은 같은 유입에서 "얼마나 잘 사게 하나"(전환·객단가)로 영향력을 본다.

실행 · 운영06

현실적 운영법

라이브에서 변수 하나만 딱 바꾸는 건 불가능하다(호스트·상품·시즌은 매번 바뀜). "완벽한 실험"은 포기하고 "기록"으로 푼다.

핵심은 "한 개만 바꾸기"가 아니라 "빠짐없이 기록하기"

  • 통제 가능한 건 의도적으로 설계 — 플친·시간대·토스·GFA는 켜고 끌 수 있다.
  • 통제 못 하는 건 빠짐없이 기록 — 호스트·상품은 기록만 해둔다.
  • 회차가 쌓이면 사후 그룹 비교 — 요인을 켠 회차 평균 vs 끈 회차 평균. 표본이 충분하면 다변량 회귀로 각 요인 순기여를 분리.

기록만 일관되면, 통제를 못 했어도 데이터가 나중에 갈라준다.

실행 · 운영08

토스 이벤트 주의

외부 트래픽 폭증 = 체리피커일 수 있음

토스 이벤트(B-2)는 외부 트래픽을 대규모로 끌어오므로 동접·시청이 폭증할 수 있으나, 이벤트 보상을 노린 체리피커 비중이 높을 확률이 크다. 트래픽 폭증 대비 인크리멘탈 매출이 안 따라오면 허수로 판단한다. 다른 요인 테스트와 섞이지 않게 분리해서 기록하고, 토스 비용은 별도 마케팅비로 ROI에 반영한다.
